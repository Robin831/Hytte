- **Async PokĂŠmon scan REST API** - Replaced the synchronous `POST /api/pokemon/scan` endpoint with an async queue. New endpoints: `POST /api/pokemon/scans/queue` (multipart upload, returns 202 immediately), `GET /api/pokemon/scans` (newest-first list with status filter), `GET /api/pokemon/scans/{id}/image` (serves the stored scan image), and `POST /api/pokemon/scans/{id}/resolve` (add to collection, discard, or retry a no_match/failed scan). The shared `upsertCollection` logic is now reusable by both the collection endpoint and the scan "add" action. (Hytte-7fgp)
