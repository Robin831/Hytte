category: Added
- **HR zone boundaries single source of truth** - Added `internal/hrzones` package with `GetDefaultZones(maxHR)` using the Olympiatoppen 5-zone percentages and `GetUserZones(db, userID)` that reads custom zone boundaries from `user_preferences` with a default fallback. Added `zone_boundaries` as a validated user preference key (JSON array of 5 `{zone, min_bpm, max_bpm}` objects, zones 1–5 each present exactly once with monotonically increasing boundaries) so users can store custom HR zone boundaries. No schema migration is required — `zone_boundaries` is stored as a new key in the existing `user_preferences` key/value table. (Hytte-h3gj)
