category: Added
- **Family Chat: voice/video call signalling relay** - Added four new SSE event types (`call_offer`, `call_answer`, `call_ice`, `call_end`) and matching `POST /api/familychat/conversations/{id}/calls/{call_id}/{offer|answer|ice|end}` endpoints that relay WebRTC signalling through the existing SSE hub without persisting SDP or ICE payloads. A new `family_chat_calls` table records the call envelope (initiator, started/ended timestamps, status) so the conversation history can render missed-call entries; the offer handler also fires a high-priority webpush notification with the tag `familychat-call-<conversation_id>` to wake recipients who are not already streaming.
