category: Added
- **Pokémon async scanner: daily cost cap and image dedupe** - `POST /api/pokemon/scans/queue` now enforces a per-user daily cap (default 600 scans/day, overridable per user via the `pokemon_scan_daily_cap` preference) and returns `429 Too Many Requests` with `{cap, used}` when exhausted. Identical uploads from the same user within 10 seconds short-circuit to the existing job id and return `202 Accepted` with a `X-Pokemon-Scan-Dedupe: true` header, so the camera auto-detect loop cannot burn vision-API budget on a card that lingers in frame. `GET /api/pokemon/scans` now includes a `today: { used, cap }` block so the frontend can render "47 / 600 today" without a second request.
