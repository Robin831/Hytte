category: Changed
- **Serve vault downloads through an `io.Reader` and set `Content-Length` from stored size** - Vault download and preview responses now write the response body via `io.Copy` from an `io.Reader` and set `Content-Length` from the stored `size_bytes` instead of `len(data)`. The decrypted file is still buffered in memory first because AES-256-GCM authenticates the whole ciphertext before any plaintext is emitted, so peak memory and time-to-first-byte are unchanged; this is a structural change that keeps `Content-Length` correct and prepares the read path for true streaming if a chunked at-rest format is adopted later.
