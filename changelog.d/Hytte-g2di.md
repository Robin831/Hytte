- **Restore homework draft when send fails** - When a homework question fails to send (network error, non-OK response, or SSE error mid-stream), the optimistic user bubble is now removed, the original question is put back into the composer, and the selected image is restored — instead of leaving an unanswered bubble in the transcript with an empty input. The recovery tracks the server-assigned user message id, so a swapped real-id bubble is also removed; if the user has typed a newer draft while the send was in flight, the newer draft is preserved.
