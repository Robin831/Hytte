- **Kiosk page no longer blank on old browsers** - Added `@vitejs/plugin-legacy` to generate a transpiled legacy bundle with Babel syntax transforms (optional chaining, nullish coalescing, etc.) and language-level ES polyfills via `core-js`/`regenerator-runtime`. Web APIs such as `fetch` and `AbortController` are not polyfilled by the legacy plugin; `AbortController` usage is already guarded in code and `fetch` errors are swallowed so the kiosk falls back to the last known data. Also added a React error boundary that displays a visible error message instead of a blank white screen if the kiosk JS fails to initialise. (Hytte-e8ei)
